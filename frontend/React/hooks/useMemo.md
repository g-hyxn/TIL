# useMemo

useMemo는 컴포넌트를 최적화 해주는 훅입니다. <br>
반복되는 계산 같은 것을 메모제이션을 활용해 메모리에 저장하여 프로그램의 속도를 빠르게 해줍니다.

```js
function Memo() {
  const a = Hi();
  return <div>{a}</div>;
}

function Hi() {
  return 10;
}
```

위 코드에서 Memo() 가 렌더링 될때마다 a 변수가 초기화 되는것을 알 수 있습니다.
렌더링이 될 때마다 Hi 함수가 실행되기 때문에 불필요한 연산이 발생합니다. 이는 함수가 복잡할 수록 더욱 비효율적입니다.

따라서 동일한 값을 반환 하는 함수가 있다면 useMemo를 사용해 값을 저장하고 필요할때마다 계산하지 않고 꺼내 쓸 수 있습니다.

## useMemo 구조

```js
const a = useMemo(() => {
  return Hi();
}, [item]);
```

useMemo는 처음 인자로 콜백함수 두번째 인자로 의존성 배열을 받습니다 <br>
의존성 배열에 있는 값이 업데이트 즉 변경된다면 콜백함수를 호출하여 그 값을 메모리에 업데이트 합니다.

만약 두번째 인자에 빈 배열을 넣는다면 처음 마운트 될 시에만 값을 계산하고 그 후엔 저장된 값을 꺼내서 사용합니다.

## 사용시 주의사항

useMemo를 아무 때나 사용하면 오히려 성능이 저하될 수 있다
메모리 저장 비용과 의존성 비교 과정에서 발생하는 비용이 추가되기 때문입니다.

따라서 useMemo는 복잡한 계산이 필요한 경우에 사용하는 것이 효율적입니다.  
또한 동일한 값을 반복해서 사용하거나, 렌더링이 자주 발생하는 상황에서 사용하면 성능 개선에 도움이 될 수 있습니다.
