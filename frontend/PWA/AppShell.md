# App Shell

## 앱의 구조

웹사이트를 렌더링 하는 방법은 기본적으로 SSR과 CSR 두가지 방법이 있습니다.
<br/>

- SSR 즉 서버 사이드 렌더링은 웹사이트가 서버에서 랜더링 되는 것을 의미 합니다. 따라서 웹 초기 방문은 빠르지만 페이지를 이동할 때마다 매번 다운로드를 해야 합니다. 각 페이지를 로딩 할때마다 서버를 거쳐야 해서 성능 적인 부분에서 어려움이 있습니다.
- CSR 즉 클라이언트 사이드 렌더링은 웹사이트가 페이지를 이동할 때 거의 즉시 렌더링 됩니다. 하지만 시작할 때 더 많은 초기 다운로드와 추가 렌더링이 필요합니다. 웹 초기 방문이 느리다는 단점이 있습니다.

SSR과 CSR을 혼합하여 웹 사이트를 렌더링 한다면 초기 접근시 SSR을 이용하여 빠르게 렌더링하고 그 이후 페이지 이동시에는 CSR을 이용하여 효율적으로 웹을 렌더링 할 수 있습니다.

PWA를 구축할 때도 위에 설명한 랜더링 방식을 사용 할 수 있습니다.
이때 사용 되는 개념이 바로 app shell 입니다.

## App Shell

app shell의 개념은 화면의 레이아웃을 먼저 불러오고 실제 데이터는 나중에 불러오는 구조 입니다.
<br/>
자주 바뀌지 않는 UI 뼈대는 한 번만 로드하고,
자주 변경되는 데이터만 네트워크로 가져와 갱신하는 방식입니다.
예를 들어 헤더, 사이드바 같은 것들이 있겠네요.

### 장점

장점으로는

- 앱이 빠르게 느껴집니다.
- 이전 값들을 캐싱하여 오프라인 환경에서도 기본 UI를 표시 할 수 있습니다.
- 데이터만 갱신하여 효율적입니다.

App Shell 구조는 앞서 설명한 SSR CSR을 혼합한 환경에서 자주 쓰입니다.
