# Vite란?

Vite(비트)는 프랑스어로 빠르다는 뜻으로 번들링없이 개발 서버를 즉시 시작하는 프론트엔드 빌드 도구 이다.

## 왜 나왔을까?

기존에는 Webpack 이라는 번들러가 많이 쓰였다고 한다. 하지만 프로젝트가 커질수록 서버를 시작할때 시간이 너무 오래걸린다는 단점이 있었다. 이유는 Webpack은 서버를 시작하기 전에 모든 파일을 하나로 묶는 **번들링** 작업을 수행하기 때문에 프로젝트가 커져서 파일이 많을수록 느릴 수 밖에 없다.

## vite는 이러한 문제를 어떻게 해결했을까?

브라우저가 ES Modules를 지원 한다는 점을 활용 했다고 한다. 번들링을 하지 않고 브라우저가 파일을 요청 할때만 변환해서 돌려주는 방식이다.<br/>
node_modules에 있는 라이브러리들은 esbuild로 미리 처리해두는데 esbuild 자체가 Go로 만들어져서 Webpack보다 10~100배 빠르다고 한다.<br/>
HMR(코드 수정시 자동 반영)도 마찬가지로 수정된 파일 하나만 교체하면 돼서 프로젝트 크기와 상관 없이 빠르게 동작 한다.

## 어떨 때 쓰면 좋은가

Next.js처럼 자체 빌드 시스템이 내장된 프레임워크를 쓴다면 Vite는 필요 없다. Vite가 주로 쓰이는 건 자체 번들러가 없는 환경인데 대표적으로 순수 React SPA 프로젝트 같은 경우다.

## 아쉬운 점

현재 대체되고 있는 추세이긴 하지만 그래도 webpack에 비해 생태계가 작다는 단점이 있다.
<br>
또한 vite는 ESM을 기반으로 작동하기 때문에 최신 브라우저 환경엔 적합하지만 오래된 브라우저를 사용해야 할때나 기존 프로젝트를 마이그레이션 할땐 좀 복잡할 수 있다는 단점이 있다.

## 설치 방법

```
npm create vite@latest 프로젝트이름
```

이후 여러 선택지가 나오는데 자신이 사용하고 싶은 라이브러리, 언어등을 적절히 선택하면 된다.
